## The Mac Problem in Aerospace

### **Why Most Tools Don't Support Mac:**

**1. Legacy Codebase**
- Many tools (AGI STK, FreeFlyer) were built 20+ years ago
- Windows-only architecture (C++/Windows APIs)
- Difficult/expensive to port to Mac

**2. Enterprise Focus**
- Target market is Windows-heavy (government, defense contractors)
- Mac users are minority in enterprise aerospace
- ROI for Mac port doesn't justify cost

**3. Technical Constraints**
- Windows-specific libraries (COM, DirectX, .NET)
- Hardware dependencies (Windows drivers)
- Licensing (Windows-only licenses)

---

### **Impact on Mac Users:**

**Research Labs:**
- **Problem:** Many researchers use Macs (30-40% market share in research)
- **Workaround:** Use Windows VMs (slow, expensive, poor experience)
- **Result:** Excluded from commercial tools (AGI STK, FreeFlyer)

**Universities:**
- **Problem:** Universities issue Macs to students/faculty
- **Workaround:** Install Windows VMs or use university Windows labs
- **Result:** Inconvenient, extra cost, poor experience

**Startups:**
- **Problem:** Many startups use Macs (modern development tools)
- **Workaround:** Buy Windows machines or use VMs
- **Result:** Extra cost, infrastructure complexity

## Installation on Mac

### **Your Product on Mac:**

**1. Install Python (if not already installed):**
```bash
# Check Python version (should be 3.9-3.12)
python3 --version

# If needed, install via Homebrew
brew install python@3.11
```

**2. Install Dependencies:**
```bash
# Clone repository
git clone [your-repo]
cd satellite-control

# Create virtual environment
python3 -m venv venv

# Activate virtual environment
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt
```

**3. Run Simulation:**
```bash
# Run CLI
python3 run_simulation.py

# Or launch dashboard
python3 -m src.satellite_control.cli dashboard
```

**4. Access Web Dashboard:**
- Open Safari or Chrome
- Navigate to `http://localhost:8501`
- Full functionality (no limitations)

**Time to First Simulation:** 5 minutes (vs. hours for Windows VMs)
